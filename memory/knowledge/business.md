# business.md — 商業背景

## 商業模式

**[Inference]** 我目前的主要商業模式是：
1. **SaaS 產品**：TeamStar（訂閱制，服務直銷 / 業務團隊）
2. **內部工具**：paopao-score、xinyao-system（服務自己的組織 IPAPA）
3. **自動化工具**：fb-poster、IPAPA CRM（提升自己的業務效率）
4. **接案**：VegeNova（外部客戶，短期專案）

**[Fact]** 主力收入目標是 TeamStar SaaS。

## TeamStar 產品策略

**[Fact]** TeamStar 定位：
- 目標客群：直銷 / MLM 業務團隊
- 核心價值：業績追蹤、廣播通訊、小組戰況
- 不擴張 CRM（刻意決策，見 DECISIONS.md）
- 路線圖：戰報卡 → 上下線可見度 → 競爭對手分析

**[Fact]** 技術棧：PWA + React + Supabase，部署 Vercel，migration 目前到 028。

## 商業判斷原則

**[Fact]** 功能是否要做，我的判斷標準：
1. 它讓用戶完成什麼工作？（JTBD）
2. 不做它，用戶會離開嗎？
3. 做了它，用戶會付更多錢嗎？

**[Inference]** 我傾向先做「讓用戶留下來」的功能，再做「讓用戶付更多」的功能。

**[Fact]** 我不喜歡在「技術上很酷但用戶感受不到」的東西上花時間。

## AI 處理商業問題的原則

**[Fact]** 商業規則（計分邏輯、角色權限、付費方案）是我的決策領域，AI 不應該自行修改，只提建議。

**[Fact]** 當 AI 不確定一個改動是否影響商業邏輯，停下來問我，不要自行判斷。

**[Inference]** 我對「節省 AI API 費用」的敏感度很高，因為我是獨立開發者，預算有限。
