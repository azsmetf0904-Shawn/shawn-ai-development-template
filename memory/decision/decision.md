# decision.md — 決策風格

## 我怎麼做決策

**[Fact]** 先建立心智模型，再決策：我需要先理解「整個系統是什麼」，才能做出好的局部決策。給我片段資訊讓我做決定，我會不舒服。

**[Fact]** 偏好二選一，不喜歡開放式選擇：「A 還是 B？」比「你覺得怎樣比較好？」對我有效十倍。選項超過三個，我會開始分心。

**[Fact]** 商業目標是最終仲裁者：技術爭論的答案通常是「哪個更快讓產品上線」或「哪個更符合用戶需求」，不是「哪個更優雅」。

**[Inference]** 我的決策速度在「清楚的框架內」很快，在「開放的白板」上很慢。所以給我框架，不給我白板。

## AI 給建議的正確格式

**有效的格式：**
```
我建議 A，因為 [一個核心原因]。
B 也可以，但 [一個主要缺點]。
要繼續 A 嗎？
```

**無效的格式：**
```
有幾種方法：
1. 你可以...（優缺點列表）
2. 另外也可以...（優缺點列表）
3. 或者...（優缺點列表）
你覺得哪個比較適合你的情況？
```

## 風險判斷

**[Fact]** 我想知道風險，但不想被風險嚇到。正確方式是：「這個做法有 X 風險，解法是 Y，可以接受嗎？」

**[Inference]** 我傾向接受「已知的可管理風險」，不喜歡「未知的不確定性」。把不確定性變成清楚的風險描述，我就可以決策。

**[Fact]** 重要決策（架構、技術選型）我想親自確認，日常決策（實作細節）我授權 AI 自行判斷。

## 決策速度

**[Inference]** 我做決策很快，但執行很慢。主要瓶頸不是想清楚，而是啟動（見 governance/blindspots.md）。

**[Fact]** 當我說「先這樣，之後再優化」，代表我接受了這個方向，不要反覆問我同一個問題。
