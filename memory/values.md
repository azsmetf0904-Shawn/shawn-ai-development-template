# values.md — 核心價值觀

## 工作觀

**[Fact]** Build > Perfect：先做出可用版本，再迭代。不追求第一次就完美。

**[Fact]** 商業邏輯優先：技術服務於業務，不反過來。一個功能如果沒有解決真實問題，再技術先進也不應該做。

**[Inference]** 我比較欣賞「用最少的技術解決最大的問題」，對「技術複雜性本身」沒有崇拜。

**[Fact]** 可複製 > 獨特：我更想建立可以複製、可以教給別人的系統，而不是只有我懂的魔法。

## 對 AI 的價值觀

**[Fact]** AI 應該主動指出問題，不只是附和。一個只說「好的，我來做」的 AI，對我沒有價值。

**[Fact]** 我信任能說「這樣做有風險，我建議 A 或 B」的 AI，不信任只問「你想怎麼做？」的 AI。

**[Inference]** 我對 AI 的期待類似「好的導師」：比我懂更多技術細節，但尊重我的商業判斷。

## 時間與精力觀

**[Fact]** 我討厭重複做同樣的事。如果一件事做了兩次，就應該自動化或建立 SOP。

**[Inference]** 我願意花大量時間建立「一次性的基礎設施」（模板、流程、規範），因為我相信這會節省未來的時間。

**[Hypothesis]** 我可能低估了「維護現有系統」的時間成本，傾向建新的而不是深化舊的。

## Shawn Rule No.1（2026-06-29 確立）

**[Fact]** 任何新工具，都必須先回答一個問題：
**「它讓我完成哪一個真正的目標？」**

不是：好不好玩 / 最新 / 很酷。
而是：它到底幫我完成哪一個產品？
