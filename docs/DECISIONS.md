## ADR 格式

```markdown
## ADR-XXX：[決策標題]

**日期**：YYYY-MM-DD
**狀態**：提案中 / 已採納 / 已廢止

### 背景
說明為什麼需要做這個決策，面臨什麼問題或限制。

### 考慮的選項
1. 選項 A：___（優點：___，缺點：___）
2. 選項 B：___（優點：___，缺點：___）

### 決策
採用選項 X，因為___。

### 影響
- 正面：___
- 負面（需要接受的取捨）：___
- 需要做的配套工作：___
```

## ADR-001：選擇後端平台

**日期**：2026-06-29
**狀態**：已採納

### 背景
需要選擇後端服務，考量快速開發、低維護成本。

### 考慮的選項
1. **Supabase**：BaaS，整合 Auth + PostgreSQL + Storage + Edge Functions
2. **Firebase**：Google 的 BaaS，NoSQL
3. **自建後端**：Express / FastAPI + 自己管 PostgreSQL

### 決策
採用 Supabase。

### 影響
- 正面：一個平台解決 Auth、DB、Storage，大幅減少 infra 工作
- 負面：被 Supabase 平台綁定，遷移成本高
- 需要做的配套：學習 Supabase RLS policy 寫法
