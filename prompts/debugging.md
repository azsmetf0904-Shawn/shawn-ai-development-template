# Prompt：除錯模式

使用時機：遇到 Bug，需要 AI 協助找出根本原因。

---

## 黃金原則：先找根因，再修問題

不要讓 AI 直接給你「解法」，而是先讓它說清楚「為什麼會這樣」。
表面 workaround 只會讓問題之後更難找。

---

## Prompt 模板

```
我遇到一個 Bug，需要你幫我找出根本原因。

## 問題描述
[描述發生了什麼，越具體越好]

## 預期行為
[說明應該要發生什麼]

## 實際行為
[說明實際上發生了什麼]

## 重現步驟
1. ___
2. ___
3. 然後看到 ___

## 錯誤訊息 / Log
```
[貼上完整錯誤訊息]
```

## 相關程式碼
```js
[貼上相關程式碼]
```

## 我已經試過
- 試過 [方法]，結果 [沒用 / 有點用但沒解決]

---

請先解釋：
1. 為什麼會發生這個問題（根本原因）
2. 為什麼之前我試過的方法沒有解決根本原因
3. 正確的修法

不要直接給我修好的程式碼，先讓我確認你的分析是對的。
```

---

## 快速版（日常使用）

```
這個 Bug 是怎麼回事？

錯誤訊息：[貼上]
發生情況：[一句話描述]
相關程式碼：[貼上關鍵部分]

先說明根本原因，再給修法。
```

## 常見除錯情境

### Supabase query 回傳 null / 空陣列

```
Supabase query 回傳空資料，但資料庫裡確實有資料。

query 程式碼：[貼上]
Supabase 錯誤（如果有）：[貼上]
RLS policy 設定：[描述]

可能是 RLS policy 問題嗎？還是 query 條件有誤？
```

### React state 更新後畫面沒有重新渲染

```
我更新了 state 但畫面沒有更新。

state 更新程式碼：[貼上]
render 的部分：[貼上]

這是直接 mutate 了 state 物件的問題嗎？還是其他原因？
```

### 非同步操作順序問題

```
這個函式有時候正常、有時候出錯，感覺是 race condition。

相關程式碼：[貼上]
什麼情況下出錯：[描述]

幫我分析 async 的執行順序，找出為什麼。
```

---

## 除錯思路（AI 沒給根因時追問）

- 「你說是 [原因]，但為什麼在 [情況 A] 下是正常的？」
- 「這個修法解決了症狀，但根本原因是什麼？下次會不會在別的地方出現？」
- 「有沒有其他地方可能因為同樣的原因出現類似問題？」
