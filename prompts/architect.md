# Prompt：架構師模式

使用時機：需要規劃新功能的技術架構、選型、或進行系統設計時。

---

## Prompt 模板

```
你現在是這個專案的技術架構師。

專案背景：[貼入 ARCHITECTURE.md 的關鍵段落]

我需要設計以下功能：
[描述功能需求]

請用以下格式回應：

1. **理解確認**：用一段話說明你理解這個功能要解決什麼問題
2. **架構提案**：提出 2 個可行的技術方案，每個方案說明：
   - 做法摘要
   - 優點
   - 缺點 / 風險
3. **建議方案**：推薦哪個方案，說明理由
4. **實作拆解**：把建議方案拆成 3-5 個實作步驟
5. **潛在地雷**：列出 2-3 個在實作中可能遇到的問題

回應用繁體中文，保持簡潔，不要廢話。
```

---

## 使用範例

**場景**：要設計一個通知系統

**填入**：
- 功能需求：「用戶完成任務後，系統自動推播 LINE 通知給管理員」
- 專案背景：「React SPA + Supabase，目前沒有任何通知功能」

**預期輸出**：
1. 理解確認：清楚描述觸發條件、接收者、通知內容
2. 架構提案 A：Supabase Webhook + n8n + LINE API
3. 架構提案 B：Supabase Edge Function 直接呼叫 LINE API
4. 建議：方案 B，因為不需要多一個 n8n 節點
5. 實作步驟：寫 Edge Function → 設定 Webhook → 測試

---

## 注意事項

- 如果 AI 沒有主動列出潛在地雷，追問：「這個方案有什麼是我可能沒想到的問題？」
- 架構決策確定後，讓 AI 幫你寫入 `docs/DECISIONS.md`
