# Prompt：程式碼審查模式

使用時機：功能實作完成後，請 Claude 審查程式碼品質與安全性。

---

## Prompt 模板

```
請審查以下程式碼，專注在以下幾個面向：

1. **安全性**：是否有 XSS、SQL injection、未驗證輸入、敏感資料洩漏等問題
2. **商業邏輯**：實作是否符合需求描述，有沒有遺漏的邊界情況
3. **效能**：是否有 N+1 query、不必要的 re-render、大量資料未分頁等問題
4. **可讀性**：命名是否清楚，邏輯是否容易理解
5. **可能的 Bug**：你覺得在什麼情況下這段程式碼會出錯

---

## 需求說明
[貼入這個功能的需求描述]

## 程式碼
[貼入要審查的程式碼]

---

請用以下格式回應：

**風險等級**：低 / 中 / 高

**發現的問題**：
1. [嚴重] [描述問題] → 建議修法：___
2. [中等] [描述問題] → 建議修法：___
3. [輕微] [描述問題] → 建議修法：___

**可以上線嗎**：可以 / 修完以下問題再上線：[列問題編號]

**好的做法**（值得保留的）：[如果有的話]
```

---

## 輕量版（快速確認）

```
幫我快速看一下這段程式碼有沒有明顯的安全問題或邏輯錯誤：

[貼入程式碼]

背景：[一句話說明這段程式碼是做什麼用的]
```

---

## 審查 checklist（自用）

在請 AI 審查前，自己先過一遍：

- [ ] 沒有 console.log 敏感資料
- [ ] 沒有 hardcode API key 或 secret
- [ ] 所有用戶輸入都有驗證
- [ ] async 函式都有 try/catch
- [ ] Supabase query 都有 error handling
- [ ] 不需要的 TODO comment 都已清除
