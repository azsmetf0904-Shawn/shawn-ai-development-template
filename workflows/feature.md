# Workflow：功能開發 SOP

從需求到上線的完整流程，適用於任何新功能開發。

---

## 流程圖

```
需求確認 → 技術設計 → 任務拆解 → 實作 → 審查 → 部署
   (Claude)    (Claude)   (Claude)  (Codex) (Claude)  (Vercel)
```

---

## Step 1：需求確認（5-15 分鐘）

**目的**：確保所有人（你 + AI）對需求的理解一致，不要跳過。

**動作**：
1. 用 `prompts/planning.md` 的模板請 Claude 確認需求
2. 確認以下問題已有答案：
   - 這個功能解決什麼問題？
   - 誰會用它？怎麼用？
   - 什麼是「完成」的標準？
   - 有沒有不能動的限制？
3. 有疑問時，選擇簡單版本先做，不要試圖一次完美

**完成標準**：你可以用一句話清楚說出「這個功能是讓[誰]可以做[什麼]，讓他[得到什麼好處]」

---

## Step 2：技術設計（10-30 分鐘）

**目的**：決定怎麼實作，避免做到一半才發現方向錯了。

**動作**：
1. 用 `prompts/architect.md` 的模板請 Claude 提出技術方案
2. 確認以下設計決策：
   - 資料庫 schema 是否需要新增欄位或表格？
   - 需要哪些新的 Supabase query？
   - UI 流程是什麼？
   - 有沒有需要呼叫 Edge Function？
3. 重要決策寫入 `docs/DECISIONS.md`

**完成標準**：有明確的技術方案，可以直接拆成任務開始做

---

## Step 3：任務拆解（5-10 分鐘）

**目的**：把大功能切成可執行的小任務，每個任務 2 小時內可完成。

**動作**：
1. 把任務寫進 `TASKS.md` 的 Backlog
2. 確認每個任務有：
   - 清楚的描述
   - 完成標準
   - 相依關係（哪個要先做）
3. 決定今天要先做哪幾個，移到 Doing

**完成標準**：`TASKS.md` Doing 區塊有 1-3 個明確任務

---

## Step 4：實作（主要工作時間）

**目的**：把設計轉成可運作的程式碼。

**動作**：

如果讓 Codex 做：
1. 用 `prompts/coding.md` 的模板撰寫任務 prompt
2. 提供：任務描述、相關檔案路徑、完成標準、限制條件
3. Codex 完成後，自己快速瀏覽確認方向正確

如果讓 Claude Code 做：
1. 直接在 Claude Code 描述任務
2. Claude Code 會主動讀取 `CLAUDE.md` 的工作規範

**每完成一個任務**：
- 在 `TASKS.md` 把任務移到 Done
- 做一次簡單的手動測試確認功能正常

**完成標準**：所有 Doing 的任務都移到 Done，功能在本地環境可以正常使用

---

## Step 5：程式碼審查（10-20 分鐘）

**目的**：在上線前找出潛在問題。

**動作**：
1. 用 `prompts/review.md` 的模板請 Claude 審查
2. 重點關注：安全性、商業邏輯正確性、邊界情況
3. 修正 [嚴重] 和 [中等] 問題
4. [輕微] 問題視情況決定

**完成標準**：沒有 [嚴重] 問題，[中等] 問題已修正或記錄在 `TASKS.md`

---

## Step 6：部署

**目的**：把功能上線。

**動作**：
```bash
# 1. commit 並 push
git add [相關檔案]
git commit -m "feat: [功能名稱] - [一句話說明]"
git push

# 2. 確認 Vercel Preview 部署成功
# 3. 在 Preview URL 手動測試主要流程
# 4. 確認沒問題後，promote 到 production
vercel --prod
```

**完成標準**：
- [ ] Production 環境功能正常
- [ ] 主要用戶流程測試完成
- [ ] `CHANGELOG.md` 更新

---

## 常見卡關點

| 卡關情況 | 解法 |
|----------|------|
| 需求不清楚 | 先做最小版本，再迭代 |
| 技術選型猶豫 | 問 Claude 給 A 或 B，不要自己想太久 |
| DB schema 改不確定 | 先在 Supabase dashboard 手動試，確認再寫 migration |
| Codex 做出來不對 | 給更具體的 prompt，特別是「完成標準」和「相關檔案」|
